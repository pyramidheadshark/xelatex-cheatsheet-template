Итоговый Конспект: Логистическая Регрессия (для Собеседования)

(Используем согласованные обозначения: mmm - число объектов, nnn - число признаков, x(i)x^{(i)}x(i) - вектор признаков iii-го объекта ([x0(i),x1(i),...,xn(i)]T[x_0^{(i)}, x_1^{(i)}, ..., x_n^{(i)}]^T[x0(i)​,x1(i)​,...,xn(i)​]T, где x0(i)=1x_0^{(i)}=1x0(i)​=1), y(i)y^{(i)}y(i) - истинный класс iii-го объекта (0 или 1), www - вектор весов ([w0,w1,...,wn]T[w_0, w_1, ..., w_n]^T[w0​,w1​,...,wn​]T), p^(i)\hat{p}^{(i)}p^​(i) - предсказанная вероятность класса 1 для iii-го объекта, J(w)J(w)J(w) - функция потерь, α\alphaα - скорость обучения).

0. Введение и Назначение

  • Определение: Логистическая регрессия – это линейный классификатор, используемый преимущественно для задач бинарной классификации. Несмотря на слово "регрессия" в названии, она моделирует вероятность принадлежности объекта к определенному классу.
  • Задача: Предсказать вероятность P(y=1∣x;w)P(y=1 | x; w)P(y=1∣x;w) – вероятность того, что объект с признаками xxx принадлежит классу 1, при заданных параметрах модели www.
  • Ключевая идея: Использовать линейную комбинацию признаков, но "пропустить" результат через специальную функцию (сигмоиду), чтобы получить значение в диапазоне (0, 1), интерпретируемое как вероятность.

1. Модель и Предсказание Вероятности

  1. Шаг 1: Линейная Комбинация (Логит)

    • Определение: Вычисляется взвешенная сумма признаков объекта iii, включая фиктивный признак x0(i)=1x_0^{(i)}=1x0(i)​=1 для учета смещения w0w_0w0​. Это значение называется логит (logit) и обозначается z(i)z^{(i)}z(i).
    • Формула (скалярная): z(i)=w0x0(i)+w1x1(i)+⋯+wnxn(i)=∑j=0nwjxj(i)z^{(i)} = w_0 x_0^{(i)} + w_1 x_1^{(i)} + \dots + w_n x_n^{(i)} = \sum_{j=0}^{n} w_j x_j^{(i)}z(i)=w0​x0(i)​+w1​x1(i)​+⋯+wn​xn(i)​=∑j=0n​wj​xj(i)​
    • Формула (векторная): z(i)=wTx(i)z^{(i)} = w^T x^{(i)}z(i)=wTx(i)
    • Смысл: Логит – это "сырой" выход линейной части модели. Он может быть любым действительным числом (−∞,+∞)(-\infty, +\infty)(−∞,+∞).

  2. Шаг 2: Сигмоидная Функция (Логистическая Функция)

    • Определение: Нелинейная функция σ(z)\sigma(z)σ(z), которая преобразует любое действительное число zzz (логит) в значение в интервале (0, 1).
    • Формула: σ(z)=11+e−z\sigma(z) = \frac{1}{1 + e^{-z}}σ(z)=1+e−z1​ (где e≈2.718e \approx 2.718e≈2.718 - основание натурального логарифма)
    • Свойства Сигмоиды (Важно для интервью):
      • Диапазон: 0<σ(z)<10 < \sigma(z) < 10<σ(z)<1 для всех zzz. Идеально для вероятностей.
      • Монотонность: Строго возрастающая функция. Большему логиту соответствует большая вероятность.
      • Гладкость: Бесконечно дифференцируема везде. Это важно для градиентного спуска.
      • Симметрия: σ(−z)=1−σ(z)\sigma(-z) = 1 - \sigma(z)σ(−z)=1−σ(z). Центрирована вокруг точки (0, 0.5).
      • Производная: σ′(z)=dσdz=σ(z)(1−σ(z))\sigma'(z) = \frac{d\sigma}{dz} = \sigma(z) (1 - \sigma(z))σ′(z)=dzdσ​=σ(z)(1−σ(z)). Простая и удобная форма производной – ключевое преимущество для вывода градиента LogLoss.
      • График: S-образная кривая.
    • Возможный вопрос: Можно ли заменить сигмоиду?
      • Ответ: Да, можно использовать другие функции, отображающие (−∞,+∞)(-\infty, +\infty)(−∞,+∞) в (0, 1) и обладающие похожими свойствами (монотонность, гладкость). Пример: пробит-функция (интегральная функция стандартного нормального распределения, CDF). Однако сигмоида приводит к более простой форме градиента для LogLoss и имеет вероятностную интерпретацию через ММП. Замена повлияет на функцию потерь и интерпретацию весов.

  3. Шаг 3: Предсказание Вероятности

    • Определение: Предсказанная вероятность p^(i)\hat{p}^{(i)}p^​(i) класса 1 для объекта iii получается применением сигмоиды к логиту z(i)z^{(i)}z(i).
    • Формула (скалярная/векторная): p^(i)=P(y(i)=1∣x(i);w)=σ(z(i))=σ(wTx(i))=11+e−∑j=0nwjxj(i)\hat{p}^{(i)} = P(y^{(i)}=1 | x^{(i)}; w) = \sigma(z^{(i)}) = \sigma(w^T x^{(i)}) = \frac{1}{1 + e^{-\sum_{j=0}^{n} w_j x_j^{(i)}}}p^​(i)=P(y(i)=1∣x(i);w)=σ(z(i))=σ(wTx(i))=1+e−∑j=0n​wj​xj(i)​1​

  4. Предсказание Класса (Решающее Правило)

    • Обычно используется порог 0.5:
      • Если p^(i)≥0.5  ⟹  \hat{p}^{(i)} \ge 0.5 \impliesp^​(i)≥0.5⟹ Предсказать класс 1.
      • Если p^(i)<0.5  ⟹  \hat{p}^{(i)} < 0.5 \impliesp^​(i)<0.5⟹ Предсказать класс 0.
    • Замечание: Порог 0.5 соответствует решению wTx(i)≥0w^T x^{(i)} \ge 0wTx(i)≥0. То есть, решающая граница логистической регрессии является линейной в пространстве признаков (гиперплоскость wTx=0w^T x = 0wTx=0).

2. Функция Потерь (Log Loss / Бинарная Кросс-Энтропия)

  1. Цель: Нужна функция J(w)J(w)J(w), которая измеряет несоответствие между предсказанными вероятностями p^(i)\hat{p}^{(i)}p^​(i) и истинными классами y(i)y^{(i)}y(i) (0 или 1), и которую мы будем минимизировать для нахождения оптимальных весов www.
  2. Почему не MSE? (Ключевой вопрос):
    • Главная причина: Если использовать MSE J=1m∑(σ(wTx(i))−y(i))2J = \frac{1}{m} \sum (\sigma(w^T x^{(i)}) - y^{(i)})^2J=m1​∑(σ(wTx(i))−y(i))2, то итоговая функция потерь J(w)J(w)J(w) оказывается невыпуклой. Это значит, что у нее может быть много локальных минимумов, и градиентный спуск не гарантирует сходимость к глобальному минимуму.
    • Дополнительно: MSE штрафует ошибки квадратично, что может быть нелогично для вероятностей; LogLoss имеет более естественную вероятностную интерпретацию.
  3. Формула Log Loss (для всей выборки):
    • Определение: Средняя по всем объектам отрицательная логарифмическая функция правдоподобия.
    • Скалярная форма: J(w)=−1m∑i=1m[y(i)log⁡(p^(i))+(1−y(i))log⁡(1−p^(i))]J(w) = -\frac{1}{m} \sum_{i=1}^{m} [ y^{(i)} \log(\hat{p}^{(i)}) + (1 - y^{(i)}) \log(1 - \hat{p}^{(i)}) ]J(w)=−m1​∑i=1m​[y(i)log(p^​(i))+(1−y(i))log(1−p^​(i))] (где log⁡\loglog - натуральный логарифм, p^(i)=σ(wTx(i))\hat{p}^{(i)} = \sigma(w^T x^{(i)})p^​(i)=σ(wTx(i)))
    • Векторная форма (с использованием поэлементных операций): J(w)=−1m[yTlog⁡(p^)+(1−y)Tlog⁡(1−p^)]J(w) = -\frac{1}{m} [\mathbf{y}^T \log(\hat{\mathbf{p}}) + (1 - \mathbf{y})^T \log(1 - \hat{\mathbf{p}})]J(w)=−m1​[yTlog(p^​)+(1−y)Tlog(1−p^​)] (где p^=σ(Xw)\hat{\mathbf{p}} = \sigma(\mathbf{X}w)p^​=σ(Xw))
  4. Интерпретация Штрафа (Как работает формула):
    • Если y(i)=1y^{(i)}=1y(i)=1: Штраф равен −log⁡(p^(i))-\log(\hat{p}^{(i)})−log(p^​(i)). Он стремится к 0 при p^(i)→1\hat{p}^{(i)} \to 1p^​(i)→1 (верно и уверенно) и к +∞+\infty+∞ при p^(i)→0\hat{p}^{(i)} \to 0p^​(i)→0 (неверно и уверенно).
    • Если y(i)=0y^{(i)}=0y(i)=0: Штраф равен −log⁡(1−p^(i))-\log(1 - \hat{p}^{(i)})−log(1−p^​(i)). Он стремится к 0 при p^(i)→0\hat{p}^{(i)} \to 0p^​(i)→0 (верно и уверенно) и к +∞+\infty+∞ при p^(i)→1\hat{p}^{(i)} \to 1p^​(i)→1 (неверно и уверенно).
    • Вывод: LogLoss сильно штрафует за уверенные ошибки.
  5. Происхождение Log Loss (Принцип Максимального Правдоподобия - MLE):
    • Предположение: Считаем, что y(i)y^{(i)}y(i) следует распределению Бернулли с параметром p=p^(i)=σ(wTx(i))p = \hat{p}^{(i)} = \sigma(w^T x^{(i)})p=p^​(i)=σ(wTx(i)). То есть, P(y(i)∣x(i);w)=(p^(i))y(i)(1−p^(i))1−y(i)P(y^{(i)} | x^{(i)}; w) = (\hat{p}^{(i)})^{y^{(i)}} (1 - \hat{p}^{(i)})^{1 - y^{(i)}}P(y(i)∣x(i);w)=(p^​(i))y(i)(1−p^​(i))1−y(i).
    • Правдоподобие (Likelihood) выборки: Так как объекты независимы, правдоподобие всей выборки L(w)=P(y∣X;w)L(w) = P(\mathbf{y} | \mathbf{X}; w)L(w)=P(y∣X;w) равно произведению вероятностей для каждого объекта: L(w)=∏i=1mP(y(i)∣x(i);w)=∏i=1m(p^(i))y(i)(1−p^(i))1−y(i)L(w) = \prod_{i=1}^{m} P(y^{(i)} | x^{(i)}; w) = \prod_{i=1}^{m} (\hat{p}^{(i)})^{y^{(i)}} (1 - \hat{p}^{(i)})^{1 - y^{(i)}}L(w)=∏i=1m​P(y(i)∣x(i);w)=∏i=1m​(p^​(i))y(i)(1−p^​(i))1−y(i)
    • Максимизация Правдоподобия: Мы хотим найти такие www, которые максимизируют L(w)L(w)L(w).
    • Логарифмирование Правдоподобия (Log-Likelihood): Максимизировать L(w)L(w)L(w) эквивалентно максимизации log⁡(L(w))\log(L(w))log(L(w)) (так как логарифм - монотонная функция). Это делают, чтобы:
      • Превратить произведение в сумму (логарифм произведения = сумма логарифмов), что гораздо проще дифференцировать.
      • Избежать численной неустойчивости при перемножении многих малых вероятностей (<1). log⁡L(w)=∑i=1m[y(i)log⁡(p^(i))+(1−y(i))log⁡(1−p^(i))]\log L(w) = \sum_{i=1}^{m} [ y^{(i)} \log(\hat{p}^{(i)}) + (1 - y^{(i)}) \log(1 - \hat{p}^{(i)}) ]logL(w)=∑i=1m​[y(i)log(p^​(i))+(1−y(i))log(1−p^​(i))]
    • Переход к минимизации: Максимизация log⁡L(w)\log L(w)logL(w) эквивалентна минимизации −log⁡L(w)-\log L(w)−logL(w). Если усреднить по выборке (разделить на mmm), получим: min⁡wJ(w)=min⁡w(−1mlog⁡L(w))\min_w J(w) = \min_w \left( -\frac{1}{m} \log L(w) \right)minw​J(w)=minw​(−m1​logL(w)) Это и есть формула Log Loss!
  6. Свойства Log Loss (Важно):
    • Выпуклость: Log Loss является выпуклой функцией относительно параметров www. Это гарантирует, что градиентный спуск найдет единственный глобальный минимум.

3. Обучение Модели: Градиентный Спуск

  1. Цель: Найти вектор весов www, минимизирующий функцию потерь J(w)J(w)J(w), итеративно обновляя веса в направлении анти-градиента.
  2. Аналитическое решение: В отличие от линейной регрессии с MSE, для логистической регрессии нет простого аналитического решения. Используются итеративные методы оптимизации.
  3. Градиент Функции Потерь Log Loss:
    • Определение: Вектор ∇J(w)\nabla J(w)∇J(w), состоящий из частных производных ∂J∂wk\frac{\partial J}{\partial w_k}∂wk​∂J​ по каждому весу wkw_kwk​ (включая w0w_0w0​).
    • Формула частной производной (скалярная): (Результат после применения chain rule и использования σ′(z)=σ(z)(1−σ(z))\sigma'(z) = \sigma(z)(1-\sigma(z))σ′(z)=σ(z)(1−σ(z))) ∂J∂wk=1m∑i=1m(p^(i)−y(i))xk(i)\frac{\partial J}{\partial w_k} = \frac{1}{m} \sum_{i=1}^{m} (\hat{p}^{(i)} - y^{(i)}) x_k^{(i)}∂wk​∂J​=m1​∑i=1m​(p^​(i)−y(i))xk(i)​ (Примечание: Формула идентична градиенту MSE, но p^(i)\hat{p}^{(i)}p^​(i) вычисляется через сигмоиду!)
    • Формула градиента (векторная): ∇J(w)=1mXT(σ(Xw)−y)\nabla J(w) = \frac{1}{m} \mathbf{X}^T (\sigma(\mathbf{X}w) - \mathbf{y})∇J(w)=m1​XT(σ(Xw)−y)
  4. Алгоритм Градиентного Спуска (Mini-Batch GD):
    • Инициализация: www (нули или малые случайные), α\alphaα, BBB, критерий остановки.
    • Цикл по Эпохам:
      • Перемешать данные.
      • Цикл по Мини-Батчам:
        • Forward Pass: Посчитать логиты z(i)=wTx(i)z^{(i)} = w^T x^{(i)}z(i)=wTx(i) и вероятности p^(i)=σ(z(i))\hat{p}^{(i)} = \sigma(z^{(i)})p^​(i)=σ(z(i)) для объектов iii батча.
        • Gradient Calculation (Backward Pass): Посчитать градиент gk=1B∑i∈Batch(p^(i)−y(i))xk(i)g_k = \frac{1}{B} \sum_{i \in \text{Batch}} (\hat{p}^{(i)} - y^{(i)}) x_k^{(i)}gk​=B1​∑i∈Batch​(p^​(i)−y(i))xk(i)​ для каждого k=0..nk=0..nk=0..n.
        • Weight Update: Обновить все веса: wk:=wk−α⋅gkw_k := w_k - \alpha \cdot g_kwk​:=wk​−α⋅gk​.
  5. Варианты GD: Batch (весь датасет, медленно, точно), Stochastic (1 объект, быстро, шумно), Mini-batch (компромисс, стандарт де-факто).
  6. Оптимизаторы: Вместо простого GD часто используют более продвинутые:
    • Momentum: Добавляет "инерцию" к обновлению весов, помогая быстрее проходить плато и сглаживать колебания.
    • RMSProp: Адаптирует скорость обучения для каждого параметра на основе скользящего среднего квадратов градиентов.
    • Adam: Сочетает идеи Momentum и RMSProp. Часто является хорошим выбором по умолчанию.
    • AdamW: Adam с корректным применением L2-регуляризации (weight decay).

4. Практические Аспекты и "Фишки" для Интервью

  1. Масштабирование Признаков (Feature Scaling):
    • Необходимо ли? Да, настоятельно рекомендуется.
    • Почему?
      • Ускорение сходимости GD: Признаки разного масштаба приводят к "вытянутой" форме функции потерь, замедляя GD. Масштабирование делает ее более "сферической".
      • Корректная работа регуляризации: L1/L2 штрафуют абсолютные значения весов. Без масштабирования штраф будет непропорционален для признаков разного масштаба.
    • Методы: StandardScaler (к N(0,1)), MinMaxScaler (к).
  2. Инициализация Весов:
    • Влияет ли? Так как LogLoss выпуклая, не влияет на конечный результат (глобальный минимум един), но влияет на скорость сходимости. Инициализация нулями обычно работает нормально для LogReg (в отличие от глубоких нейросетей).
  3. Регуляризация (L1/L2):
    • Может и должна применяться к логистической регрессии для борьбы с переобучением (особенно при большом числе признаков).
    • Добавляется к функции потерь LogLoss (штрафуются веса w1,...,wnw_1, ..., w_nw1​,...,wn​, но не w0w_0w0​).
    • Механизмы и эффекты L1 (разреженность, отбор признаков) и L2 (сжатие весов) аналогичны линейной регрессии.
  4. Интерпретируемость:
    • Веса wjw_jwj​ в логистической регрессии интерпретируются сложнее, чем в линейной. Они показывают, как изменяется логит (логарифм отношения шансов log(p1−p)log(\frac{p}{1-p})log(1−pp​)) при изменении xjx_jxj​ на единицу. Часто смотрят на exp(wj)exp(w_j)exp(wj​) - во сколько раз изменятся шансы (p/(1−p)p/(1-p)p/(1−p)).
  5. Сравнение с Другими Моделями:
    • Простота и Эффективность: LogReg – простая, быстрая, часто дает хороший baseline.
    • Линейная Граница: Основное ограничение – может разделять классы только линейно. Если граница сложная, нужны другие модели (деревья, SVM с ядром, нейросети) или полиномиальные признаки.
  6. Мультиклассовая Логистическая Регрессия:
    • Использует Softmax функцию на выходе вместо сигмоиды.
    • Функция потерь – Cross-Entropy Loss.
    • Может рассматриваться как однослойная нейронная сеть с Softmax активацией.

5. Краткое Резюме (Ключевые Точки для Запоминания)

  • Задача: Бинарная классификация (предсказание вероятности P(y=1∣x)P(y=1|x)P(y=1∣x)).
  • Модель: p^=σ(wTx)\hat{p} = \sigma(w^T x)p^​=σ(wTx), где σ(z)=1/(1+e−z)\sigma(z) = 1 / (1 + e^{-z})σ(z)=1/(1+e−z) (Сигмоида).
  • Loss: Log Loss (Бинарная Кросс-Энтропия), выпуклая, выводится из ММП.
  • Обучение: Градиентный спуск (или его варианты), нет аналитического решения.
  • Градиент: ∂J∂wk=1m∑(p^(i)−y(i))xk(i)\frac{\partial J}{\partial w_k} = \frac{1}{m} \sum (\hat{p}^{(i)} - y^{(i)}) x_k^{(i)}∂wk​∂J​=m1​∑(p^​(i)−y(i))xk(i)​.
  • Практика: Нуждается в масштабировании признаков, можно применять L1/L2 регуляризацию.
  • Ограничение: Линейная решающая граница.
