Часть 1: Дерево Решений (Decision Tree)

1.1 Критерии "Чистоты" Узла (Impurity Measures) - Классификация

Критерии чистоты используются для оценки того, насколько хорошо сплит разделяет данные на однородные группы в дочерних узлах. Цель - найти сплит, максимизирующий уменьшение нечистоты (Information Gain).

1.1.1 Gini Impurity (Индекс Джини)

Определение: Вероятность того, что случайно выбранный элемент из узла будет неправильно классифицирован, если ему присвоить случайный класс согласно распределению классов в этом узле.

Формула:

G=∑k=1Kpk(1−pk)G = \sum_{k=1}^{K} p_k (1 - p_k) G=k=1∑K​pk​(1−pk​)

  • pkp_kpk​: Доля объектов класса kkk в текущем узле. Рассчитывается как Nk/NnodeN_k / N_{node}Nk​/Nnode​, где NkN_kNk​ - число объектов класса kkk в узле, NnodeN_{node}Nnode​ - общее число объектов в узле.
  • (1−pk)(1 - p_k)(1−pk​): Вероятность того, что объект не принадлежит классу kkk.
  • pk(1−pk)p_k (1 - p_k)pk​(1−pk​): Вероятность выбрать объект класса kkk и ошибочно присвоить ему другой класс (согласно распределению). Суммирование по всем классам дает общую вероятность ошибки.

Альтернативная (более частая) формула:

G=∑k=1Kpk(1−pk)=∑k=1K(pk−pk2)=∑k=1Kpk−∑k=1Kpk2\begin{aligned} G &= \sum_{k=1}^{K} p_k (1 - p_k) \\ &= \sum_{k=1}^{K} (p_k - p_k^2) \\ &= \sum_{k=1}^{K} p_k - \sum_{k=1}^{K} p_k^2 \\ \end{aligned} G​=k=1∑K​pk​(1−pk​)=k=1∑K​(pk​−pk2​)=k=1∑K​pk​−k=1∑K​pk2​​

Так как ∑k=1Kpk=1\sum_{k=1}^{K} p_k = 1∑k=1K​pk​=1 (сумма долей всех классов равна 1), получаем:

G=1−∑k=1Kpk2\boxed{ G = 1 - \sum_{k=1}^{K} p_k^2 } G=1−k=1∑K​pk2​​

  • Объяснение: Эта форма эквивалентна предыдущей. Она вычитает из 1 сумму квадратов долей классов. Чем "чище" узел (доминирует один класс), тем больше один из pkp_kpk​ близок к 1, а остальные к 0, и тем больше ∑pk2\sum p_k^2∑pk2​ близка к 1, а GGG близка к 0.
  • Пример:
    • Узел с классами {0: 10, 1: 0} (чистый): p0=1,p1=0p_0=1, p_1=0p0​=1,p1​=0. G=1−(12+02)=0G = 1 - (1^2 + 0^2) = 0G=1−(12+02)=0.
    • Узел с классами {0: 5, 1: 5} (макс. нечистота для K=2): p0=0.5,p1=0.5p_0=0.5, p_1=0.5p0​=0.5,p1​=0.5. G=1−(0.52+0.52)=1−(0.25+0.25)=0.5G = 1 - (0.5^2 + 0.5^2) = 1 - (0.25 + 0.25) = 0.5G=1−(0.52+0.52)=1−(0.25+0.25)=0.5.

1.1.2 Entropy (Энтропия Шеннона)

Определение: Мера неопределенности или "хаоса" в распределении классов в узле, основанная на теории информации. Измеряет, сколько в среднем "бит" информации нужно для кодирования класса случайно выбранного объекта из узла.

Формула:

H=−∑k=1Kpklog⁡2(pk)\boxed{ H = - \sum_{k=1}^{K} p_k \log_2(p_k) } H=−k=1∑K​pk​log2​(pk​)​

  • pkp_kpk​: Доля объектов класса kkk в узле.
  • log⁡2(pk)\log_2(p_k)log2​(pk​): Логарифм по основанию 2. Часто используются и другие основания (натуральный log), это меняет масштаб энтропии, но не положение максимума/минимума, поэтому на выбор лучшего сплита не влияет.
  • Соглашение: Если pk=0p_k = 0pk​=0, то считаем pklog⁡2(pk)=0p_k \log_2(p_k) = 0pk​log2​(pk​)=0.
  • Объяснение: Член −pklog⁡2(pk)-p_k \log_2(p_k)−pk​log2​(pk​) представляет "информационное содержание" наблюдения класса kkk. Сумма по всем классам дает ожидаемое информационное содержание (энтропию). Энтропия равна 0 для чистого узла (pk=1p_k=1pk​=1 для одного kkk) и максимальна при равномерном распределении классов (pk=1/Kp_k = 1/Kpk​=1/K для всех kkk).
  • Пример:
    • Узел {0: 10, 1: 0}: p0=1,p1=0p_0=1, p_1=0p0​=1,p1​=0. H=−[1log⁡2(1)+0⋅log⁡2(0)]=−[1⋅0+0]=0H = - [1 \log_2(1) + 0 \cdot \log_2(0)] = - [1 \cdot 0 + 0] = 0H=−[1log2​(1)+0⋅log2​(0)]=−[1⋅0+0]=0.
    • Узел {0: 5, 1: 5}: p0=0.5,p1=0.5p_0=0.5, p_1=0.5p0​=0.5,p1​=0.5. H=−[0.5log⁡2(0.5)+0.5log⁡2(0.5)]=−[0.5⋅(−1)+0.5⋅(−1)]=−[−0.5−0.5]=1H = - [0.5 \log_2(0.5) + 0.5 \log_2(0.5)] = - [0.5 \cdot (-1) + 0.5 \cdot (-1)] = - [-0.5 - 0.5] = 1H=−[0.5log2​(0.5)+0.5log2​(0.5)]=−[0.5⋅(−1)+0.5⋅(−1)]=−[−0.5−0.5]=1.

1.2 Критерий "Чистоты" Узла - Регрессия

1.2.1 Variance Reduction (Уменьшение Дисперсии)

Определение: В задачах регрессии целью является минимизация разброса (дисперсии) значений целевой переменной yyy внутри листьев. Критерием чистоты узла служит сама дисперсия yyy в этом узле.

Формула Дисперсии в Узле:

Varnode=1Nnode∑i∈node(y(i)−yˉnode)2\boxed{ Var_{node} = \frac{1}{N_{node}} \sum_{i \in \text{node}} (y^{(i)} - \bar{y}_{node})^2 } Varnode​=Nnode​1​i∈node∑​(y(i)−yˉ​node​)2​

  • NnodeN_{node}Nnode​: Число объектов в текущем узле.
  • y(i)y^{(i)}y(i): Истинное значение целевой переменной для iii-го объекта в узле.
  • yˉnode\bar{y}_{node}yˉ​node​: Среднее значение yyy по всем объектам в узле (yˉnode=1Nnode∑i∈nodey(i)\bar{y}_{node} = \frac{1}{N_{node}} \sum_{i \in \text{node}} y^{(i)}yˉ​node​=Nnode​1​∑i∈node​y(i)). Это значение обычно и является предсказанием дерева в данном узле, если он станет листом.
  • Объяснение: Формула измеряет средний квадрат отклонения значений yyy от их среднего в узле. Это то же самое, что MSE (Среднеквадратичная ошибка), если бы мы предсказывали среднее значение для всех объектов в узле. Цель сплита - найти такое разделение, чтобы суммарная (взвешенная) дисперсия в дочерних узлах была как можно меньше.

1.3 Критерий Качества Сплита: Information Gain (IG)

Определение: Насколько уменьшилась нечистота (Gini, Entropy) или дисперсия (Variance) после выполнения сплита по сравнению с родительским узлом. Мы выбираем сплит, который максимизирует этот прирост.

Формула:

IG(Parent, Split)=Impurity(Parent)−∑c∈ChildrenNcNParentImpurity(Child c)\boxed{ IG(\text{Parent, Split}) = \text{Impurity}(\text{Parent}) - \sum_{c \in \text{Children}} \frac{N_c}{N_{\text{Parent}}} \text{Impurity}(\text{Child } c) } IG(Parent, Split)=Impurity(Parent)−c∈Children∑​NParent​Nc​​Impurity(Child c)​

  • Impurity(Parent)\text{Impurity}(\text{Parent})Impurity(Parent): Значение критерия нечистоты (Gini, Entropy или Variance) в родительском узле (до сплита).
  • Children\text{Children}Children: Множество дочерних узлов, полученных в результате сплита (обычно два: левый и правый).
  • Impurity(Child c)\text{Impurity}(\text{Child } c)Impurity(Child c): Значение критерия нечистоты в дочернем узле ccc.
  • NcN_cNc​: Число объектов, попавших в дочерний узел ccc.
  • NParentN_{\text{Parent}}NParent​: Общее число объектов в родительском узле (NParent=∑NcN_{\text{Parent}} = \sum N_cNParent​=∑Nc​).
  • NcNParent\frac{N_c}{N_{\text{Parent}}}NParent​Nc​​: Доля объектов, ушедших в дочерний узел ccc.
  • ∑c∈ChildrenNcNParentImpurity(Child c)\sum_{c \in \text{Children}} \frac{N_c}{N_{\text{Parent}}} \text{Impurity}(\text{Child } c)∑c∈Children​NParent​Nc​​Impurity(Child c): Средневзвешенное значение нечистоты в дочерних узлах.
  • Объяснение: Мы вычитаем из исходной нечистоты родителя средневзвешенную нечистоту детей. Чем больше это значение (IG), тем "чище" стали дочерние узлы в среднем, и тем лучше данный сплит.

1.4 Cost Complexity Pruning (CCP)

Определение: Метод обрезки дерева после его построения для борьбы с переобучением путем минимизации функционала, штрафующего за сложность дерева (число листьев).

Функционал Стоимости-Сложности:

Rα(T)=R(T)+α∣T∣\boxed{ R_\alpha(T) = R(T) + \alpha |T| } Rα​(T)=R(T)+α∣T∣​

  • TTT: Рассматриваемое дерево или поддерево.
  • R(T)R(T)R(T): Суммарная ошибка или нечистота листьев дерева TTT. Рассчитывается как сумма нечистот (Gini, Entropy, MSE) по всем листьям дерева: R(T)=∑l∈Leaves(T)Nl⋅Impurity(l)R(T) = \sum_{l \in \text{Leaves}(T)} N_l \cdot \text{Impurity}(l)R(T)=∑l∈Leaves(T)​Nl​⋅Impurity(l), где NlN_lNl​ - число объектов в листе lll.
  • ∣T∣|T|∣T∣: Количество листьев в дереве TTT. Мера сложности дерева.
  • α≥0\alpha \ge 0α≥0: Параметр сложности (гиперпараметр). Контролирует баланс между точностью (R(T)R(T)R(T)) и сложностью (∣T∣|T|∣T∣).
    • При α=0\alpha=0α=0, R0(T)=R(T)R_0(T) = R(T)R0​(T)=R(T), выбирается самое большое (необрезанное) дерево.
    • При увеличении α\alphaα, штраф за сложность растет, и алгоритм предпочитает обрезать больше ветвей, выбирая меньшие поддеревья.
  • Объяснение: Алгоритм ищет такое поддерево TTT, которое минимизирует Rα(T)R_\alpha(T)Rα​(T) для заданного α\alphaα. Варьируя α\alphaα, можно получить последовательность вложенных оптимально обрезанных деревьев. Оптимальное значение ccp_alpha для использования в модели подбирается по кросс-валидации.

Часть 2: Ансамблевые Методы

2.1 Bagging (Bootstrap Aggregating)

Определение: Метод ансамблирования, где BBB базовых моделей обучаются на BBB бутстрэп-выборках (выборки с возвращением из исходных данных), а их предсказания агрегируются.

Агрегация Предсказаний: Пусть hb(x)h_b(x)hb​(x) - предсказание bbb-й базовой модели (обученной на bbb-й бутстрэп-выборке) для объекта xxx. Итоговое предсказание ансамбля H(x)H(x)H(x):

  • Регрессия: Среднее арифметическое.

    H(x)=1B∑b=1Bhb(x)\boxed{ H(x) = \frac{1}{B} \sum_{b=1}^{B} h_b(x) } H(x)=B1​b=1∑B​hb​(x)​

  • Классификация (Голосование большинством):

    H(x)=argmaxk∈{1,...,K}∑b=1BI(hb(x)=k)\boxed{ H(x) = \text{argmax}_{k \in \{1, ..., K\}} \sum_{b=1}^{B} \mathbb{I}(h_b(x) = k) } H(x)=argmaxk∈{1,...,K}​b=1∑B​I(hb​(x)=k)​

    • I(⋅)\mathbb{I}(\cdot)I(⋅) - индикаторная функция (1, если условие верно, 0 иначе). Мы выбираем класс kkk, который был предсказан наибольшим числом базовых моделей.
  • Классификация (Усреднение вероятностей): Если базовые модели hb(x)h_b(x)hb​(x) возвращают вектор вероятностей классов [pb1(x),...,pbK(x)][p_{b1}(x), ..., p_{bK}(x)][pb1​(x),...,pbK​(x)], то:

    Вероятности ансамбля: p^k(x)=1B∑b=1Bpbk(x)\text{Вероятности ансамбля: } \hat{p}_k(x) = \frac{1}{B} \sum_{b=1}^{B} p_{bk}(x) Вероятности ансамбля: p^​k​(x)=B1​b=1∑B​pbk​(x)

    H(x)=argmaxk∈{1,...,K}p^k(x)\boxed{ H(x) = \text{argmax}_{k \in \{1, ..., K\}} \hat{p}_k(x) } H(x)=argmaxk∈{1,...,K}​p^​k​(x)​

    • Объяснение: Усреднение вероятностей часто дает более гладкие и надежные результаты, чем жесткое голосование.

2.2 Random Forest (RF)

RF использует Bagging для деревьев решений с добавлением Random Subspace. Формулы агрегации предсказаний те же, что и для Bagging (см. выше). Основные формулы, специфичные для RF, касаются оценки важности признаков.

2.2.1 Mean Decrease in Impurity (MDI) / Gini Importance

Логика Расчета (Не единая формула, а алгоритм):

  1. Для каждого дерева TbT_bTb​ в ансамбле:
    • Для каждого внутреннего узла vvv в дереве TbT_bTb​:
      • Если для сплита в узле vvv использовался признак jjj:
        • Рассчитать уменьшение нечистоты (Impurity Decrease), достигнутое этим сплитом: ΔI(v)=Nv⋅Impurity(v)−(Nv,L⋅Impurity(vL)+Nv,R⋅Impurity(vR))\Delta I(v) = N_v \cdot \text{Impurity}(v) - (N_{v,L} \cdot \text{Impurity}(v_L) + N_{v,R} \cdot \text{Impurity}(v_R))ΔI(v)=Nv​⋅Impurity(v)−(Nv,L​⋅Impurity(vL​)+Nv,R​⋅Impurity(vR​)). (Где NNN - число объектов в узлах).
        • Добавить ΔI(v)\Delta I(v)ΔI(v) к суммарному вкладу признака jjj для дерева TbT_bTb​.
  2. Усреднить суммарный вклад каждого признака jjj по всем деревьям BBB.
  3. Нормализовать полученные важности (например, чтобы их сумма была равна 1).
  • Объяснение: MDI оценивает, насколько в среднем каждый признак помогал уменьшить нечистоту узлов при построении деревьев на обучающей выборке.

2.2.2 Permutation Importance / Mean Decrease in Accuracy (MDA)

Логика Расчета (Алгоритм):

  1. Обучить ансамбль RF.
  2. Рассчитать базовую метрику качества SbaseS_{base}Sbase​ на OOB-выборке (или валидационной/тестовой).
  3. Для каждого признака jjj (от 1 до nnn): a. Сохранить копию OOB-выборки (или val/test). b. Перемешать значения только jjj-го столбца (признака jjj) в этой копии. c. Сделать предсказания с помощью обученного RF на данных с перемешанным jjj-м признаком. d. Рассчитать метрику качества SjS_jSj​ на этих предсказаниях. e. Рассчитать важность признака jjj: Impj=Sbase−SjImp_j = S_{base} - S_jImpj​=Sbase​−Sj​.
  4. (Опционально) Повторить шаги 3.b-3.e несколько раз и усреднить ImpjImp_jImpj​ для стабильности.
  5. (Опционально) Нормализовать важности ImpjImp_jImpj​.
  • Объяснение: MDA оценивает, насколько падает качество модели на "новых" данных, если разорвать связь между признаком jjj и целевой переменной путем его перемешивания. Чем больше падение, тем важнее признак.

2.3 Gradient Boosting Machines (GBM)

2.3.1 Обновление Предсказания Ансамбля

Определение: На каждой итерации mmm предсказание ансамбля Fm(x)F_m(x)Fm​(x) обновляется добавлением предсказания новой базовой модели hm(x)h_m(x)hm​(x) (обученной на псевдо-остатках), умноженного на скорость обучения ν\nuν.

Формула:

Fm(x)=Fm−1(x)+ν⋅hm(x)\boxed{ F_m(x) = F_{m-1}(x) + \nu \cdot h_m(x) } Fm​(x)=Fm−1​(x)+ν⋅hm​(x)​

  • Fm(x)F_m(x)Fm​(x): Предсказание ансамбля после добавления mmm-й модели.
  • Fm−1(x)F_{m-1}(x)Fm−1​(x): Предсказание ансамбля на предыдущем шаге.
  • ν\nuν: Скорость обучения (learning rate, eta). Малое положительное число (0 < ν\nuν <= 1).
  • hm(x)h_m(x)hm​(x): Предсказание mmm-й базовой модели (дерева).
  • Объяснение: Мы делаем небольшой шаг (ν⋅hm(x)\nu \cdot h_m(x)ν⋅hm​(x)) в направлении, которое предсказывает mmm-я модель (направлении анти-градиента), чтобы скорректировать предсказание ансамбля.

2.3.2 Вычисление Псевдо-Остатков (Targets для hmh_mhm​)

Определение: Целевая переменная для обучения mmm-й базовой модели hm(x)h_m(x)hm​(x). Она равна отрицательному градиенту функции потерь LLL, вычисленному по предсказанию ансамбля на предыдущем шаге Fm−1F_{m-1}Fm−1​.

Общая Формула:

rim=−[∂L(yi,F(xi))∂F(xi)]F(xi)=Fm−1(xi)\boxed{ r_{im} = - \left[ \frac{\partial L(y_i, F(x_i))}{\partial F(x_i)} \right]_{F(x_i) = F_{m-1}(x_i)} } rim​=−[∂F(xi​)∂L(yi​,F(xi​))​]F(xi​)=Fm−1​(xi​)​​

  • rimr_{im}rim​: Псевдо-остаток для iii-го объекта на mmm-й итерации.
  • L(yi,F(xi))L(y_i, F(x_i))L(yi​,F(xi​)): Дифференцируемая функция потерь для iii-го объекта.
  • ∂L∂F\frac{\partial L}{\partial F}∂F∂L​: Частная производная функции потерь по ее второму аргументу (предсказанию модели FFF).
  • [… ]F=Fm−1[\dots]_{F=F_{m-1}}[…]F=Fm−1​​: Означает, что производная вычисляется в точке, где предсказание равно предсказанию ансамбля на предыдущем шаге Fm−1(xi)F_{m-1}(x_i)Fm−1​(xi​).

Примеры для Конкретных Функций Потерь:

  • MSE (для Регрессии):

    • L(yi,F)=12(yi−F)2L(y_i, F) = \frac{1}{2}(y_i - F)^2L(yi​,F)=21​(yi​−F)2
    • ∂L∂F=12⋅2(yi−F)⋅(−1)=−(yi−F)=F−yi\frac{\partial L}{\partial F} = \frac{1}{2} \cdot 2 (y_i - F) \cdot (-1) = -(y_i - F) = F - y_i∂F∂L​=21​⋅2(yi​−F)⋅(−1)=−(yi​−F)=F−yi​
    • rim=−[Fm−1(xi)−yi]=yi−Fm−1(xi)r_{im} = - [F_{m-1}(x_i) - y_i] = y_i - F_{m-1}(x_i)rim​=−[Fm−1​(xi​)−yi​]=yi​−Fm−1​(xi​)
    • Объяснение: Для MSE псевдо-остаток - это просто обычный остаток (разница между истинным значением и предсказанием предыдущего ансамбля).

  • Log Loss (для Бинарной Классификации):

    • FFF здесь - это логит (выход до сигмоиды), p=σ(F)p = \sigma(F)p=σ(F).
    • L(yi,p)=−[yilog⁡(p)+(1−yi)log⁡(1−p)]L(y_i, p) = -[y_i \log(p) + (1-y_i)\log(1-p)]L(yi​,p)=−[yi​log(p)+(1−yi​)log(1−p)]
    • Нужна производная по FFF: ∂L∂F=∂L∂p⋅∂p∂F\frac{\partial L}{\partial F} = \frac{\partial L}{\partial p} \cdot \frac{\partial p}{\partial F}∂F∂L​=∂p∂L​⋅∂F∂p​.
    • Мы выводили ранее: ∂L∂p=−yi−pp(1−p)\frac{\partial L}{\partial p} = -\frac{y_i - p}{p(1-p)}∂p∂L​=−p(1−p)yi​−p​ и ∂p∂F=σ′(F)=p(1−p)\frac{\partial p}{\partial F} = \sigma'(F) = p(1-p)∂F∂p​=σ′(F)=p(1−p).
    • ∂L∂F=−yi−pp(1−p)⋅p(1−p)=−(yi−p)=p−yi\frac{\partial L}{\partial F} = -\frac{y_i - p}{p(1-p)} \cdot p(1-p) = -(y_i - p) = p - y_i∂F∂L​=−p(1−p)yi​−p​⋅p(1−p)=−(yi​−p)=p−yi​
    • rim=−[pm−1(xi)−yi]=yi−pm−1(xi)r_{im} = - [p_{m-1}(x_i) - y_i] = y_i - p_{m-1}(x_i)rim​=−[pm−1​(xi​)−yi​]=yi​−pm−1​(xi​)
      • Где pm−1(xi)=σ(Fm−1(xi))p_{m-1}(x_i) = \sigma(F_{m-1}(x_i))pm−1​(xi​)=σ(Fm−1​(xi​)) - предсказанная вероятность на предыдущем шаге.
    • Объяснение: Для Log Loss псевдо-остаток - это разница между истинной меткой (0 или 1) и предсказанной вероятностью предыдущего ансамбля. Новое дерево hm(x)h_m(x)hm​(x) будет учиться предсказывать это расхождение, чтобы скорректировать логит F(x)F(x)F(x).
