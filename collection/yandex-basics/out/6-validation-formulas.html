1. K-Fold Cross-Validation: Оценка Метрики

Цель: Получить более надежную оценку качества модели, усреднив результаты по нескольким разбиениям данных.

Шаг 1: Разбиение Данных Данные (Train+Validation) делятся на KKK непересекающихся фолдов.

Шаг 2: Итеративное Обучение и Оценка Для каждого фолда kkk (от 1 до KKK):

  1. Обучить модель на данных из K−1K-1K−1 фолдов (всех, кроме kkk-го).
  2. Рассчитать метрику качества MkM_kMk​ на kkk-м (валидационном) фолде.

Шаг 3: Усреднение Результатов Итоговая оценка метрики (Mˉ\bar{M}Mˉ) и ее стандартное отклонение (std(M)std(M)std(M)) рассчитываются как:

Mˉ=1K∑k=1KMk\boxed{ \bar{M} = \frac{1}{K} \sum_{k=1}^{K} M_k } Mˉ=K1​k=1∑K​Mk​​

std(M)=1K−1∑k=1K(Mk−Mˉ)2(Выборочное ст. отклонение)\boxed{ std(M) = \sqrt{\frac{1}{K-1} \sum_{k=1}^{K} (M_k - \bar{M})^2} \quad (\text{Выборочное ст. отклонение}) } std(M)=K−11​k=1∑K​(Mk​−Mˉ)2​(Выборочное ст. отклонение)​

  • Объяснение: Mˉ\bar{M}Mˉ дает среднюю оценку производительности модели. std(M)std(M)std(M) показывает, насколько стабильна эта оценка при разных разбиениях данных (низкое значение std(M)std(M)std(M) предпочтительнее).

2. Масштабирование Признаков (Препроцессинг)

Цель: Привести признаки к одному масштабу, что важно для алгоритмов, чувствительных к масштабу (GD, KNN, модели с регуляризацией). fit делается только на обучающих данных (train), transform применяется ко всем (train, val, test).

2.1 StandardScaler Приводит данные к нулевому среднему (μ=0\mu=0μ=0) и единичному стандартному отклонению (σ=1\sigma=1σ=1).

Шаг 1: Вычисление μj\mu_jμj​ и σj\sigma_jσj​ (на Train Set) Для каждого признака jjj (от 1 до nnn) вычислить среднее μj\mu_jμj​ и стандартное отклонение σj\sigma_jσj​ только по обучающей выборке:

μj=1mtrain∑i∈trainxj(i)\mu_j = \frac{1}{m_{train}} \sum_{i \in train} x_j^{(i)} μj​=mtrain​1​i∈train∑​xj(i)​

σj=1mtrain−1∑i∈train(xj(i)−μj)2(Если σj=0, использовать 1)\sigma_j = \sqrt{\frac{1}{m_{train}-1} \sum_{i \in train} (x_j^{(i)} - \mu_j)^2} \quad (\text{Если } \sigma_j = 0, \text{ использовать } 1) σj​=mtrain​−11​i∈train∑​(xj(i)​−μj​)2​(Если σj​=0, использовать 1)

Шаг 2: Применение Масштабирования (Transform) Для каждого объекта iii и каждого признака jjj:

xj,scaled(i)=xj(i)−μjσj\boxed{ x_{j, scaled}^{(i)} = \frac{x_j^{(i)} - \mu_j}{\sigma_j} } xj,scaled(i)​=σj​xj(i)​−μj​​​

  • Объяснение: Из каждого значения вычитается среднее по признаку и делится на стандартное отклонение по признаку (вычисленные на трейне).

2.2 MinMaxScaler Приводит данные к заданному диапазону, обычно [0, 1].

Шаг 1: Вычисление min⁡j\min_jminj​ и max⁡j\max_jmaxj​ (на Train Set) Для каждого признака jjj (от 1 до nnn) найти минимальное min⁡j\min_jminj​ и максимальное max⁡j\max_jmaxj​ значение только по обучающей выборке:

min⁡j=min⁡i∈train(xj(i))\min_j = \min_{i \in train} (x_j^{(i)}) jmin​=i∈trainmin​(xj(i)​)

max⁡j=max⁡i∈train(xj(i))\max_j = \max_{i \in train} (x_j^{(i)}) jmax​=i∈trainmax​(xj(i)​)

Шаг 2: Применение Масштабирования (Transform) Для каждого объекта iii и каждого признака jjj:

xj,scaled(i)=xj(i)−min⁡jmax⁡j−min⁡j(Если max⁡j=min⁡j, результат 0 или 0.5)\boxed{ x_{j, scaled}^{(i)} = \frac{x_j^{(i)} - \min_j}{\max_j - \min_j} \quad (\text{Если } \max_j = \min_j, \text{ результат 0 или 0.5}) } xj,scaled(i)​=maxj​−minj​xj(i)​−minj​​(Если jmax​=jmin​, результат 0 или 0.5)​

  • Объяснение: Значение масштабируется пропорционально его положению между минимумом и максимумом (вычисленными на трейне).

3. SMOTE (Synthetic Minority Over-sampling Technique)

Цель: Сбалансировать классы путем генерации синтетических примеров минорного класса. Применяется только к обучающей части фолда внутри CV.

Алгоритм Генерации Одного Синтетического Примера:

Шаг 1: Выбор Базового Объекта Выбрать случайный объект xix_ixi​ из минорного класса.

Шаг 2: Поиск Соседей Найти kkk ближайших соседей объекта xix_ixi​ среди объектов минорного же класса.

Шаг 3: Выбор Соседа для Интерполяции Случайно выбрать одного из этих kkk соседей, обозначим его xzix_{zi}xzi​.

Шаг 4: Генерация Синтетического Объекта Вычислить новый вектор признаков xnewx_{new}xnew​ как линейную интерполяцию между xix_ixi​ и xzix_{zi}xzi​:

xnew=xi+λ⋅(xzi−xi)\boxed{ x_{new} = x_i + \lambda \cdot (x_{zi} - x_i) } xnew​=xi​+λ⋅(xzi​−xi​)​

Где:

  • xix_ixi​ - вектор признаков базового объекта.
  • xzix_{zi}xzi​ - вектор признаков выбранного соседа.
  • λ\lambdaλ - случайное число, выбранное равномерно из диапазона [0, 1].
  • Операции сложения и умножения векторов выполняются поэлементно.

Шаг 5: Повторение Повторять шаги 1-4 до тех пор, пока не будет сгенерировано желаемое количество синтетических примеров минорного класса.

  • Объяснение: SMOTE создает новые точки на линиях, соединяющих существующие минорные точки с их ближайшими минорными соседями. Это позволяет увеличить представленность минорного класса, не просто дублируя существующие точки.

4. Взвешивание Классов (Class Weighting)

Цель: Модифицировать функцию потерь, чтобы ошибки на объектах редкого (минорного) класса наказывались сильнее, заставляя модель уделять им больше внимания.

Пример: Модификация Log Loss для Бинарной Классификации

Исходная Log Loss:

J(w)=−1m∑i=1m[y(i)log⁡(p^(i))+(1−y(i))log⁡(1−p^(i))]J(w) = -\frac{1}{m} \sum_{i=1}^{m} [ y^{(i)} \log(\hat{p}^{(i)}) + (1 - y^{(i)}) \log(1 - \hat{p}^{(i)}) ] J(w)=−m1​i=1∑m​[y(i)log(p^​(i))+(1−y(i))log(1−p^​(i))]

Взвешенная Log Loss: Добавляются веса w0w_0w0​ (для класса 0) и w1w_1w1​ (для класса 1), которые обратно пропорциональны частотам классов.

Jweighted(w)=−1m∑i=1mwy(i)[y(i)log⁡(p^(i))+(1−y(i))log⁡(1−p^(i))]\boxed{ J_{weighted}(w) = -\frac{1}{m} \sum_{i=1}^{m} w_{y^{(i)}} [ y^{(i)} \log(\hat{p}^{(i)}) + (1 - y^{(i)}) \log(1 - \hat{p}^{(i)}) ] } Jweighted​(w)=−m1​i=1∑m​wy(i)​[y(i)log(p^​(i))+(1−y(i))log(1−p^​(i))]​

Где wy(i)w_{y^{(i)}}wy(i)​ - вес, соответствующий истинному классу y(i)y^{(i)}y(i) объекта iii.

  • Если y(i)=1y^{(i)} = 1y(i)=1, используется вес w1w_1w1​.
  • Если y(i)=0y^{(i)} = 0y(i)=0, используется вес w0w_0w0​.

Расчет Весов (например, class_weight='balanced' в sklearn): Веса обычно рассчитываются обратно пропорционально частоте класса:

wk=mK⋅mk\boxed{ w_k = \frac{m}{K \cdot m_k} } wk​=K⋅mk​m​​

Где:

  • wkw_kwk​ - вес для класса kkk.

  • mmm - общее число объектов.

  • KKK - общее число классов (здесь K=2K=2K=2).

  • mkm_kmk​ - число объектов класса kkk.

  • Объяснение: Если класс kkk редкий (mkm_kmk​ мало), его вес wkw_kwk​ будет большим. Ошибки на объектах этого класса будут умножаться на больший вес в функции потерь, что заставит алгоритм оптимизации (GD) сильнее стараться их исправить.

5. Статистическая Значимость (Концептуальные Формулы)

Цель: Формально определить, является ли наблюдаемая разница в метриках между моделями A и B случайной или систематической.

5.1 Доверительный Интервал (CI) для Разности Средних (Пример) При сравнении средних метрик MˉA\bar{M}_AMˉA​ и MˉB\bar{M}_BMˉB​, полученных по фолдам CV, можно построить CI для разности Δ=MˉA−MˉB\Delta = \bar{M}_A - \bar{M}_BΔ=MˉA​−MˉB​.

Общая структура CI (для уровня доверия 1−α1-\alpha1−α):

CI(Δ)=Δˉ±tcrit⋅SE(Δˉ)\boxed{ CI(\Delta) = \bar{\Delta} \pm t_{crit} \cdot SE(\bar{\Delta}) } CI(Δ)=Δˉ±tcrit​⋅SE(Δˉ)​

Где:

  • Δˉ=MˉA−MˉB\bar{\Delta} = \bar{M}_A - \bar{M}_BΔˉ=MˉA​−MˉB​ - наблюдаемая средняя разность.

  • SE(Δˉ)SE(\bar{\Delta})SE(Δˉ) - стандартная ошибка оценки средней разности (зависит от стандартных отклонений и числа фолдов, для парных данных считается по разностям MA,k−MB,kM_{A,k} - M_{B,k}MA,k​−MB,k​).

  • tcritt_{crit}tcrit​ - критическое значение t-распределения (или z-распределения для больших выборок) для уровня α\alphaα и степеней свободы.

  • Интерпретация: Если CI не содержит 0, то разница статистически значима на уровне α\alphaα.

5.2 P-value из Статистического Теста (Концепция) Применяется статистический тест (например, парный t-тест) к разностям метрик dk=MA,k−MB,kd_k = M_{A,k} - M_{B,k}dk​=MA,k​−MB,k​.

Нулевая гипотеза H0H_0H0​: Средняя разность равна нулю (μd=0\mu_d = 0μd​=0). Альтернативная гипотеза H1H_1H1​: Средняя разность не равна нулю (μd≠0\mu_d \ne 0μd​=0).

Результат теста: P-value.

P-value=P(наблюдать данные как у нас или более экстремальные∣H0 верна)\boxed{ \text{P-value} = P(\text{наблюдать данные как у нас или более экстремальные} | H_0 \text{ верна}) } P-value=P(наблюдать данные как у нас или более экстремальные∣H0​ верна)​

  • Интерпретация: Если P-value < α\alphaα (заданный уровень значимости, обычно 0.05), то отвергаем H0H_0H0​. Разница статистически значима.

(Примечание: Детальный вывод формул стандартной ошибки и тестовых статистик выходит за рамки базового обзора валидации, но важно понимать концепцию CI и p-value).
